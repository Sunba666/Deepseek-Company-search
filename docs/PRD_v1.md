# PRD v1 — JobNav AI：AI 求职导航 + 内推聚合平台

> 版本：v1.1（根据优化意见修订）  
> 状态：已确认，进入用户流程图阶段  
> 一句话定位：帮助求职者更快发现岗位、获取真实有效的内推信息、智能匹配职位，并一键跳转企业官网完成投递。

---

## 1. 产品定位

**产品名称**：JobNav AI  
**产品类型**：AI 驱动的智能求职导航平台（AI Job Navigation Platform）  
**产品原则**：

| 原则 | 说明 |
|------|------|
| 🏛️ **官网优先** | 所有岗位最终都跳转企业官方招聘网站进行投递 |
| 🤖 **AI 优先** | 每一个页面都应该有 AI 提供分析，而不仅仅是展示数据 |
| ⚡ **实时优先** | 岗位、内推、招聘状态尽可能保持实时更新 |
| ✅ **可信优先** | 每条内推都应展示可信度评分，并说明评分依据 |
| 🎯 **效率优先** | 任何核心操作尽量控制在三次点击以内完成 |

**核心边界（不做清单）**：
- ❌ 不做招聘信息发布平台
- ❌ 不做企业 ATS（招聘管理系统）
- ❌ 不取代企业官网投递
- ✅ 专注于「发现岗位 + 聚合内推 + AI 分析 + 官网导航」

**核心价值**：让 AI 贯穿整个求职流程，提高求职效率。

## 14. SEO 与性能优化

### SEO 策略

| 措施 | 实现方式 |
|------|---------|
| SSR 服务端渲染 | Next.js getServerSideProps / getStaticProps |
| Sitemap 自动生成 | next-sitemap 插件，每日自动更新 |
| Robots.txt | 动态生成，区分环境 |
| Open Graph | 每个页面自定义 OG 标题/描述/图片 |
| Schema.org 结构化数据 | JobPosting / Organization / BreadcrumbList 结构化标记 |
| 语义化 URL | /jobs/react-developer-shanghai-123 而非 /jobs?id=123 |
| 规范标签 | rel=canonical 防止重复内容 |
| 移动优先 | 响应式设计 + Mobile First 索引 |

### 性能优化

| 指标 | 目标 | 措施 |
|:----:|:----:|------|
| FCP | < 1.5s | 首屏 SSR、关键 CSS 内联、字体预加载 |
| LCP | < 2.5s | 图片优化、CDN、组件懒加载 |
| FID | < 100ms | 代码分割、JS 按需加载 |
| CLS | < 0.1 | 图片占位、骨架屏、字体显示交换 |
| TTI | < 3s | 资源预加载、Service Worker 缓存 |
| API 响应 | < 200ms | Redis 缓存、数据库索引、查询优化 |

### 缓存策略

```
+------------------------------------------------------------------+
|                      缓存分层                                     |
|                                                                   |
|  1. 浏览器缓存:      静态资源（JS/CSS/字体）-> 长期缓存（1年）     |
|  2. CDN 缓存:        页面 HTML -> 短时缓存（5-60s）               |
|  3. Redis 缓存:      API 响应 -> TTL 动态（热门数据 30s-5min）    |
|  4. 数据库查询缓存:   高频查询结果缓存（岗位列表/公司列表）         |
|  5. 内存缓存:        配置信息、字典数据 -> 应用启动加载            |
+------------------------------------------------------------------+
```

## 15. 安全要求

| 安全措施 | 实现方式 |
|---------|---------|
| JWT 登录认证 | Access Token + Refresh Token 双令牌机制 |
| OAuth 登录 | GitHub / Google / 微信等第三方登录 |
| CSRF 防护 | CSRF Token + SameSite Cookie |
| XSS 防护 | 输出编码、CSP 策略、DOMPurify |
| SQL 注入防护 | TypeORM / Prisma 参数化查询 |
| API 限流 | Redis 令牌桶 + 用户/IP 级别限流 |
| 权限校验 | RBAC Guard + 接口级权限注解 |
| 操作日志 | AOP 切面记录所有写操作 |
| 敏感数据加密 | 密码 bcrypt、Token RSA、数据 AES-256 |
| HTTPS 全站 | TLS 1.3、HSTS、安全 Header |

### 16.2 代码质量要求

- **高可维护性**：单一职责、最少知识原则、依赖倒置
- **高可读性**：命名规范、注释完整、代码即文档
